Operational Maintenance: Metric Lifecycle

1. Creation Standards

  • Cardinality impact assessment required
  • Clear naming convention enforced
  • Business value documented
  • Tied to an SLO/SLI (if applicable)

2. Performance Monitoring

  • Regular overhead measurement
  • CPU & Memory cost tracking
  • Query performance analysis
  • Optimization opportunities flagged

3. Deprecation Strategy

  • Identify unused/stale metrics
  • 60-day deprecation notice period
  • Automated removal from code
  • Prevent cardinality "drift"

4. Cost Tracking

  • Monitor storage costs (TSDB)
  • Track network egress costs
  • Measure compute for aggregation
  • Attribute costs back to teams
